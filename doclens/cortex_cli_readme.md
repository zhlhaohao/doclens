# Cortex CLI 使用手册

## 简介

Cortex 是一个交互式全文检索工具，基于 TreeSearch 库，支持对 `E:/github/notebook` 目录下的文档进行快速搜索。

## 快速开始

```bash
python -m cortex
```

## 命令列表

### 搜索命令

| 命令 | 别名 | 说明 | 搜索方式 | 搜索范围 | 示例 |
|------|------|------|----------|----------|------|
| `/s <关键词>` | `/search` | 关键词搜索 | FTS5 → LIKE → ripgrep（自动降级） | 索引 + 磁盘 | `/s python` |
| `/grep <正则>` | `/g` | 正则搜索索引原文 | SQLite REGEXP | 索引原文 | `/grep 量子.*密码` |
| `/rg <正则>` | `/ripgrep` | 正则搜索磁盘文件 | ripgrep regex | 磁盘文件 | `/rg ^#{1,3}\s+部署` |
| `<关键词>` | — | 等同 `/s` | 同 `/s` | 索引 + 磁盘 | `python` |

### `/s` 搜索降级链

当 FTS5 因分词问题找不到结果时，自动逐级降级：

```
FTS5 BM25（分词索引，速度最快）
  → 严格过滤（多词邻近度评分）
    → 放松匹配（≥1 词匹配）
      → SQLite LIKE（索引原文子串匹配，不受分词影响）
        → ripgrep（磁盘文件搜索，最终兜底）
```

### 正则搜索示例

```
> /grep 量子.*密码         # SQLite 正则匹配索引中的原文
> /g ^import\\s+os         # /g 是 /grep 的别名
> /rg def\\s+test_         # ripgrep 正则搜索磁盘文件
> /ripgrep class\\w+       # /ripgrep 是 /rg 的全称
```

### 索引命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `/index` | 重建索引 | `/index` |
| `/index -f` | 强制重建索引（删除旧索引） | `/index -f` |

## 搜索结果说明

### 显示格式

```
+-- [序号] 标题
|    文件: 相对路径
|    ---------------------------------------------
|  >>> 关键词所在行（高亮显示）
|      上下文行（上下3行）
|    评分: x.xxxx
```

### 标记说明

- `>>>` - 关键词所在行
- `   ` - 上下文行
- `[1;31m...[0m` - 关键词高亮（在支持 ANSI 的终端中显示为红色）
- **文件名超链接** - 在支持 ANSI 超链接的终端（Windows Terminal、VSCode Terminal）中可点击，点击后在 VSCode 中打开对应文件

## 配置文件

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DEFAULT_SEARCH_PATH` | `E:/github/notebook` | 搜索路径 |
| `DEFAULT_INDEX_PATH` | `~/.cortex/index.db` | 索引文件路径 |

修改配置文件位置：
```python
# 在 notebook_search.py 中
DEFAULT_SEARCH_PATH = "你的笔记本目录路径"
DEFAULT_INDEX_PATH = "~/.cortex/你的索引文件路径.db"
```

## 索引管理

### 索引位置

索引文件默认保存在 `~/.cortex/index.db`

### 强制重建索引

当文档有更新时，使用：

```
> /index -f
```

这会删除旧索引并重新索引所有文件。

### 增量索引

工具支持增量索引，只会重新索引有变化的文件。除非使用 `/index -f` 强制重建。

## 常见问题

### Q: 为什么搜索结果没有关键词高亮？

A: 高亮标记 `[1;31m...[0m` 是 ANSI 转义序列，需要在支持 ANSI 的终端中查看效果。

### Q: 如何搜索多个关键词？

A: 目前支持单关键词搜索，如需多关键词可多次搜索。

### Q: 索引文件可以移动吗？

A: 可以，移动后修改 `DEFAULT_INDEX_PATH` 指向新位置即可。
