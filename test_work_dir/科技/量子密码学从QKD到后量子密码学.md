# 量子密码学：从QKD到后量子密码学

## 量子威胁

量子计算机对现有密码体系构成根本性威胁。Shor算法可以在多项式时间内分解大整数和计算离散对数，这意味着广泛使用的RSA、ECC（椭圆曲线密码）和DH（Diffie-Hellman）密钥交换将不再安全。

### 威胁时间表
- **当前**：量子计算机约1000逻辑量子比特，尚不能破解实用密码
- **2028-2030年**：预计达到足够破解RSA-2048的量子计算能力
- **"先收集后解密"攻击**：攻击者现在截获加密数据，等待量子计算机成熟后解密

因此，对长期机密性要求高的数据（国防、医疗、金融），量子威胁已经是现实问题。

## 量子密钥分发（QKD）

### 基本原理
量子密钥分发利用量子力学的基本原理实现无条件安全的密钥协商：

1. **量子不可克隆定理**：未知的量子态不能被完美复制
2. **测量坍缩**：对量子态的测量会不可逆地改变其状态
3. **海森堡不确定性原理**：不能同时精确测量互补的物理量

任何窃听行为都会引入可检测的误码率，使通信双方能够发现窃听。

### BB84协议
最经典的QKD协议，由Charles Bennett和Gilles Brassard于1984年提出：

1. Alice随机选择比特值（0或1）和基（+基或x基），发送对应的光子偏振态
2. Bob随机选择测量基进行测量
3. 通过经典信道公开比较基的选择，丢弃基不匹配的结果
4. 对剩余比特进行误码率检测，确认无窃听
5. 通过隐私放大生成最终的安全密钥

### QKD系统性能

| 参数 | 当前水平 | 2027年目标 |
|------|----------|-----------|
| 光纤传输距离 | 500-800公里 | 1000公里 |
| 密钥生成速率 | 1-10 kbps | 100+ kbps |
| 网络规模 | 城域网 | 广域网 |
| 成本/端 | $50-100万 | $10-20万 |

### 中国QKD网络
中国在QKD领域处于全球领先地位，潘建伟院士团队是核心推动力量：

- **京沪干线**：全长2000余公里的量子保密通信骨干网，由潘建伟团队主导建设
- **墨子号卫星**：实现1200公里星地量子密钥分发，潘建伟担任首席科学家
- **济南量子网**：全球首个量子保密通信城域网
- **量子通信产业联盟**：包括国盾量子、科大国创等企业

### QKD的局限性
1. **需要专用硬件**：不能在现有网络基础设施上直接部署
2. **传输距离受限**：光纤损耗导致密钥率随距离指数衰减
3. **设备不完美**：实际设备与理论模型的差异可能被利用
4. **成本高昂**：大规模部署的经济性尚待验证

## 后量子密码学（PQC）

### 与QKD的区别
- **QKD**：基于物理原理的密钥分发，需要量子信道
- **PQC**：基于数学难题的密码算法，运行在经典计算机上，可软件部署

PQC是更实用的过渡方案，因为它与现有网络基础设施兼容。

### NIST标准化进程

美国国家标准与技术研究院（NIST）的后量子密码标准化工作历时8年，于2024年正式发布了第一批标准：

#### 已发布标准

**ML-KEM（FIPS 203）— 基于格的密钥封装**
- 原名：CRYSTALS-Kyber
- 安全基础：模块格上的带错误学习（MLWE）问题
- 公钥大小：1,568字节（Kyber-1024）
- 密文大小：1,568字节
- 性能：密钥生成约15微秒，封装约20微秒
- 用途：替代RSA和ECDH密钥交换

**ML-DSA（FIPS 204）— 基于格的数字签名**
- 原名：CRYSTALS-Dilithium
- 安全基础：模块格上的短整数解（MSIS）和MLWE问题
- 公钥大小：2,592字节（Dilithium Level 5）
- 签名大小：4,627字节
- 性能：签名约150微秒，验证约50微秒
- 用途：替代RSA和ECDSA数字签名

**SLH-DSA（FIPS 205）— 无状态哈希签名**
- 原名：SPHINCS+
- 安全基础：哈希函数的抗碰撞性（不依赖格假设）
- 公钥大小：64字节
- 签名大小：29,792字节（SPHINCS+-256f）
- 性能：签名约50毫秒（较慢）
- 用途：作为备选方案，不依赖格的安全性假设

#### 候选标准
- **FALCON**：基于NTRU格的签名方案，签名更小但实现更复杂
- **BIKE/HQC**：基于纠错码的KEM方案，作为备选

### 中国后量子密码进展
中国密码管理部门也在加速推进后量子密码标准化：

- 国密局发布了基于格的密码算法草案（GM/T 00XX-2025）
- 中国科学院、清华大学等在基于编码和多变量的密码方案上有深入研究
- 金融行业率先试点后量子密码迁移
- 计划在2026-2028年完成核心系统的算法替换

## 迁移策略

### 混合部署
在过渡期采用传统算法和后量子算法并行运行的混合方案：

```
传统方案：RSA-2048 + ML-KEM-1024 → 联合密钥协商
签名方案：ECDSA-P256 + ML-DSA-65 → 联合签名验证
```

混合方案确保即使一种算法被破解，安全性仍然由另一种算法保障。

### 迁移路线图

| 阶段 | 时间 | 任务 |
|------|------|------|
| 评估 | 2025 | 盘点所有使用密码的系统，评估迁移优先级 |
| 测试 | 2025-2026 | 在测试环境部署PQC，验证兼容性和性能 |
| 试点 | 2026-2027 | 高优先级系统（金融、政务）率先迁移 |
| 推广 | 2027-2028 | 全面部署混合方案 |
| 完成 | 2028-2030 | 去除传统算法，仅使用PQC |

### 迁移挑战
1. **性能影响**：PQC算法的密钥和签名更大，网络带宽增加2-10倍
2. **互操作性**：新旧系统需要共存，协议需要支持算法协商
3. **嵌入式设备**：物联网设备的计算和存储资源有限
4. **证书体系**：PKI证书需要更新以支持更大的密钥和签名
5. **成本**：全球密码迁移预计花费数百亿美元
