📐 评分原理（白话版）
最终得分（0~1）= 把下面 5 个信号按权重做加权平均（每个信号名对应下方一个"XX 权重"字段）：
• 关键词匹配 —— 文档里命中的关键词数 ÷ 你查询的总词数
• 文件名匹配 —— 文件名里命中的关键词数 ÷ 总词数
• FTS 原始分 —— FTS5 全文检索给的相关度（0~1 之间）
• 标题匹配 —— 段落标题里命中的关键词数 ÷ 总词数
• 邻近度 —— 0 / 0.5 / 1 三档（多词紧挨着分数更高）
每个权重越大，对应信号对最终排序的影响越大；权重设为 0 = 完全忽略该信号。推荐区间 0~10。修改后下次查询即时生效。

⚖️ 权重配置

所有权重推荐区间 0 ~ 10；0 = 禁用该信号。

关键词匹配权重 ● 即时
CORTEX_WEIGHT_KEYWORD_MATCH · 范围 0~10
权重越大，越偏好"命中的关键词数量多"的文档（多关键词 query 时尤其重要）。
文件名匹配权重 ● 即时
CORTEX_WEIGHT_FILE_NAME_MATCH · 范围 0~10
权重越大，文件名包含关键词的文档排序越靠前。
FTS 原始分权重 ● 即时
CORTEX_WEIGHT_FTS_SCORE · 范围 0~10
权重越大，越偏向传统全文检索 BM25 排序（与关键词匹配度正相关）。
标题匹配权重 ● 即时
CORTEX_WEIGHT_TITLE_MATCH · 范围 0~10
权重越大，节点标题（如 Markdown 小节标题）含关键词时排序越靠前。
邻近度权重 ● 即时
CORTEX_WEIGHT_PROXIMITY_MATCH · 范围 0~10
权重越大，多关键词在文档中紧邻出现的文档越受偏好。
⚠️ 这些参数仅影响 cortex 命令行 / TUI 的终端输出格式，对 Web UI 没有可见效果。在此处提供编辑仅为了免去手动改 .env 的麻烦。

🖥️ 终端结果显示

仅影响 cortex CLI/TUI 在终端里展示搜索结果时的上下文行数。

上下文行数上限
CORTEX_MAX_CONTEXT_LINES
行
每个命中行向上/向下最多各显示多少行原文上下文。
锚点行数上限
CORTEX_MAX_ANCHOR_LINES
行
从同一文档的所有命中行里，挑出多少个"最佳行"作为展示中心（锚点）。锚点越多，结果越完整但输出越长。
锚点上下文扩展范围
CORTEX_CONTEXT_EXPAND_RANGE
行
以每个锚点为中心，向前/向后各展开多少行作为上下文（再与"上下文行数上限"取较小值）。
