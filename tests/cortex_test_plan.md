## 测试用例

### 一、索引功能 (Indexing)

#### TC-1.1: 首次索引构建

- **前置**: `e:\test\.cortex\` 目录为空（无 index.db）
- **操作**: 启动 cortex，执行 `/index`
- **预期**:
  - 显示索引进度信息
  - 输出包含：索引文件数、跳过文件数、修剪路径数、总文档数、耗时
  - `e:\test\.cortex\index.db` 文件被创建
  - 文件大小 > 0

#### TC-1.2: 增量索引（未修改文件应被跳过）

- **前置**: TC-1.1 已完成
- **操作**: 不修改任何文件，再次执行 `/index`
- **预期**:
  - 所有文件显示为 "跳过"（unchanged）
  - 索引文件数为 0（无新增）
  - 总文档数不变

#### TC-1.3: 增量索引（修改文件应被重新索引）

- **前置**: TC-1.1 已完成
- **操作**: 修改 `sample.md` 的内容（追加几行），执行 `/index`
- **预期**:
  - 修改的文件被重新索引
  - 其他文件被跳过
  - 总文档数不变（或按实际变化）

#### TC-1.4: 增量索引（新增文件）

- **前置**: TC-1.1 已完成
- **操作**: 在 `e:\test` 中新建一个 `.md` 文件，执行 `/index`
- **预期**:
  - 新文件被索引
  - 已有文件被跳过
  - 总文档数增加

#### TC-1.5: 强制重建索引

- **前置**: TC-1.1 已完成
- **操作**: 执行 `/index --force`
- **预期**:
  - 删除旧索引，全部文件重新索引
  - 无文件被跳过
  - 总文档数与首次索引一致

#### TC-1.6: 索引空目录

- **前置**: 使用一个空目录作为 search_path
- **操作**: 执行 `/index`
- **预期**:
  - 不崩溃，输出 0 文件被索引
  - index.db 被创建（但内容为空）

#### TC-1.7: 多格式文件索引

- **前置**: `e:\test` 中包含 .md, .txt, .json, .yaml, .py, .html, .pdf, .docx 等文件
- **操作**: 执行 `/index`，再执行 `/stats` 查看文件类型分布
- **预期**:
  - 所有支持格式的文件都被正确索引
  - `/stats` 的文件类型分布中包含所有格式

### 二、搜索功能 (Search)

#### TC-2.1: 基本中文关键词搜索

- **前置**: 索引已构建（包含中文 Markdown 文件）
- **操作**: `/s 孙杨`
- **预期**:
  - 返回包含 "孙杨" 关键词的搜索结果
  - 结果中关键词被高亮显示
  - 显示文件路径和行号
  - 显示上下文片段

#### TC-2.2: 多关键词搜索

- **前置**: 索引已构建
- **操作**: `/s 高铁 哈尔滨`
- **预期**:
  - 返回同时包含或邻近包含两个关键词的结果
  - 邻近匹配的结果排名更靠前（proximity_score 更高）

#### TC-2.3: 英文关键词搜索

- **前置**: 索引已构建（包含 Python/JS 代码文件）
- **操作**: `/s function`
- **预期**:
  - 返回代码文件中包含 "function" 的结果

#### TC-2.4: 搜索无结果的关键词

- **前置**: 索引已构建
- **操作**: `/s xyznonexistent12345`
- **预期**:
  - 先尝试 FTS5 搜索
  - 若 FTS5 无结果，回退到 ripgrep 精确搜索
  - 若仍无结果，显示无匹配提示
  - 不崩溃

#### TC-2.5: 搜索结果格式验证

- **前置**: 索引已构建
- **操作**: `/s 导弹`
- **预期**:
  - 标题被截断到 55 字符以内
  - 文件路径显示为 VSCode 链接格式
  - 匹配行前有 `>>>` 标记
  - 行长度超过 78 字符时被截断
  - 每个文档最多显示 3 个匹配点

#### TC-2.6: 搜索结果排序

- **前置**: 索引已构建
- **操作**: 搜索一个在多个文档中出现的关键词
- **预期**:
  - 结果按 proximity_score 降序排列
  - 同分时按 matched_count 降序排列

#### TC-2.7: PDF 文件搜索

- **前置**: 索引包含 PDF 文件
- **操作**: `/s 研发项目`
- **预期**:
  - 返回 PDF 文件中的匹配内容
  - 显示文件路径

#### TC-2.8: DOCX 文件搜索

- **前置**: 索引包含 DOCX 文件
- **操作**: 搜索 DOCX 文件中的关键词
- **预期**:
  - 返回 Word 文档中的匹配内容

#### TC-2.9: 代码文件搜索

- **前置**: 索引包含 .py 和 .js 文件
- **操作**: `/s import` 或 `/s def`
- **预期**:
  - 返回代码文件中的匹配内容
  - 显示正确的行号

#### TC-2.10: 搜索未索引状态

- **前置**: `e:\test\.cortex\` 为空（无索引）
- **操作**: 直接执行 `/s 测试`
- **预期**:
  - 自动触发索引构建
  - 索引完成后执行搜索
  - 返回搜索结果

#### TC-2.11: 空查询搜索

- **前置**: 索引已构建
- **操作**: `/s` （无参数）
- **预期**:
  - 提示需要输入搜索词，或优雅处理不崩溃

### 四、命令解析与交互

#### TC-4.1: 命令别名

- **操作**: 依次测试命令别名
  - `/q`, `/quit`, `/exit`, `/e` → 退出
  - `/h`, `/help`, `/?` → 帮助
  - `/st`, `/stats`, `/status`, `/t` → 状态
  - `/i`, `/index`, `/reindex` → 索引
  - `/s`, `/search` → 搜索
  - `/n`, `/set` → 设置
  - `/cls`, `/clear`, `/cl` → 清屏
- **预期**: 每个别名都能正确触发对应功能

#### TC-4.2: 中文顿号前缀转换

- **操作**: 输入 `、s 测试`

  - 返回搜索结果

#### TC-4.3: 未知命令

- **操作**: `/unknowncmd`
- **预期**:
  - 显示 "Unknown command" 提示
  - 不崩溃

#### TC-4.4: 空命令

- **操作**: 只输入 `/`
- **预期**:
  - 显示 "Command incomplete" 提示
  - 不崩溃

#### TC-4.5: 空输入

- **操作**: 直接按 Enter（空行）
- **预期**:
  - 不崩溃
