# 知识库引文规范设计

## 背景

当前 `cortex/skills/knowledge_base/SKILL.md` 已在「注意事项」段落中提到：

> 引用文档内容时，标注来源文件名和路径

但**没有明确规定引文的具体格式**（行内 / 末尾编号 / 脚注），也未说明去重、粒度、触发条件。LLM Agent 在使用此技能时，最终答案经常不包含任何引文来源，丢失了可追溯性。

## 目标

- 所有基于工具调用（`search_kb` / `read_document` / `grep`）生成的答案，**必须在末尾追加「## 参考资料」列表**
- 格式：编号 + 文档路径（不含 hierarchy / 行号）
- 同一文档被多处引用时**只列一次**
- 强制实施，**无配置项**

## 方案

### 1. SKILL.md 末尾追加「引文要求（强制）」段落

**文件**：`cortex/skills/knowledge_base/SKILL.md`

在「FTS 查询策略」段落之前（即「注意事项」段落后）追加一节 `## 引文要求（强制）`：

```markdown
## 引文要求（强制）

凡是答案内容来自工具调用（`search_kb` / `read_document` / `grep`）的，
**必须在末尾追加"## 参考资料"列表**。

**格式**：

- 末尾加 `## 参考资料` 二级标题
- 每条以 `1. `、`2. ` 编号（不要用 `[1]` 写在每条前）
- 内容为搜索结果 `<result>` 中 `<path>` 标签的原始值（**只取文档路径，不含 `<hierarchy>`**）
- 同一文档被多处引用时**只列一次**，保持出现顺序

**正文标注**：

- 正文中需要引用的位置插入 `[N]`（N 是参考资料编号）
- 紧跟相关陈述之后（标点前）
- 一处可同时引用多个来源：`[1][2]`

**正确示例**：

> 量子计算利用量子比特进行信息处理，具有叠加和纠缠特性 [1][2]。
>
> ## 参考资料
>
> 1. 量子计算导论/第一章.md
> 2. 量子计算导论/第二章.md

**错误示例**（禁止）：

> 量子计算利用量子比特进行信息处理。— 没有标注
>
> ## 参考资料
>
> - 第一章 — 没有完整路径
> - 第二章 > 2.1 — 包含了 hierarchy
> - [1] 第一章.md — 在条目前写编号
```

### 2. 路径来源

LLM 从工具返回的 XML 文本中提取路径：

| 工具 | 提取位置 |
|------|---------|
| `search_kb` | 每个 `<result>` 块中的 `<path>` 标签值 |
| `read_document` | 工具返回开头的 `文档:` 行后的路径 |
| `grep` | 每条结果的 `<path>` 标签值 |

### 3. 行为边界

**强制场景**：

- 答案中包含从工具返回的具体事实、数字、定义、引文 → **必须**加参考资料
- 答案仅是流程性回复（"已索引" / "已重建"）→ 可不加

**粒度**：仅文档路径，不含 `<hierarchy>`、不含行号。

**去重**：同一文档在多个 `<result>` 中出现时，参考资料列表只列一次。

## 不改动的部分

- `cortex/kb_tools.py`：不修改 `search_kb` 等工具的输出格式
- `cortex/agent_integration.py`：不修改 Agent 集成代码
- `cortex/config.py`：不增加环境变量配置项
- `search_kb` / `read_document` / `grep` 工具 schema：保持不变

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `cortex/skills/knowledge_base/SKILL.md` | 末尾追加「引文要求（强制）」段落 |

## 验证方法

**功能测试**（手动）：

1. 启动 TUI 模式（`python -m cortex`），对知识库提问 → 检查最终答案末尾是否含 `## 参考资料` 段落
2. 同一文档被多处引用 → 检查是否去重
3. 问管理类问题（"重建索引" / "统计索引"）→ 应无参考资料段落
4. 通过 `search_kb` 提问 → 检查每条路径是否来自搜索结果的 `<path>` 字段

**回归测试**：

- 已有的 `tests/test_case.json` 应当继续通过
- 知识库相关测试用例（SEARCH-001, AI-002, WATCH-003）的输出应包含 `## 参考资料` 段落
